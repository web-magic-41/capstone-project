Use case

    Benjamin wants to browse the site to try and find cards he wants to buy or trade for.

User Story

    As a Browser I Want to be able to see and search cards seamlessly, so I can find cards I may be interested in.

Precondition

Wants to browse cards to trade and buy locally online.

Post Condition

Potentially finds a card that is liked and messages to see if it is for trade or sale.

Interaction Flow

  • Ben types the url for the site.
  • The site returns the home page.
  • Ben decides not to log in since he is just browsing around for now and searches for a card.
  • The page returns the search results.
  • ben scrolls down till he finds a suitable listing.
  • the page loads the listing page.
  • Ben clicks the Dm button.
  • The site loads the DM page for the user who made the listing.
  • Ben sets up the meeting place and trades cards with the person.
  • The end (:
