Use case

    Benjamin wants to browse the site to try and find cards he wants to buy or trade for.

User Story

    As a Browser I Want to be able to see and search cards seamlessly, so I can find cards I may be interested in.

Precondition

Wants to browse cards to trade and buy locally online.

Post Condition

Potentially finds a card that is liked and messages to see if it is for trade or sale.

Interaction Flow

  • Ben types the url for the site.
  • The site returns the home page.
  • Ben decides not to log in since he is just browsing around for now.
  • Ben clicks on the search bar on the nav bar at the top of the screen
  • the site opens a field for the user to type the name of a card
  • Ben types the name of a card and hits enter.
  • The sites loads up the search results based on his search terms.
  • Ben scrolls down.
  • The site gradually loads more listings as he goes down the page.
  • Ben clicks on a listing to look at it.
  • The site returns the listing that ben clicked on.
  • Ben is now done looking at cards.
  • The end (:
